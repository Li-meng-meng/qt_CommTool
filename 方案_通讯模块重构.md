# 通讯模块重构方案

## 一、问题分析

### 1.1 当前问题

当前 `CommViewModel.cpp` 承担了过多职责：
- 串口参数管理
- 蓝牙参数管理
- 通讯模式切换逻辑
- 数据发送/接收处理
- 界面绑定逻辑

随着新增通讯方式（如蓝牙 BLE、TCP、UDP），该文件会持续膨胀，难以维护。

### 1.2 关于 BaseCommService 的思考

当前设计采用继承模式：

```cpp
class SerialService : public BaseCommService
class BluetoothService : public BaseCommService
```

**这种设计的意义：**
- 统一接口规范（`getConnectState()`、`sendData()`、`closePort()`）
- 多态调用：可以通过 `BaseCommService*` 操作不同通讯方式
- 便于扩展新的通讯方式

**当前未充分利用的原因：**
- `CommViewModel` 直接持有具体类（`SerialService`、`BluetoothService*`）
- 未使用多态，而是通过 `m_commMode` 手动切换

## 二、解决方案

### 方案 A：使用多态 + 策略模式（推荐）

```
ViewModel/
├── Comm/
│   ├── CommManager.h          # 通讯管理器（核心）
│   ├── CommManager.cpp
│   ├── SerialCommHandler.h    # 串口处理器
│   ├── SerialCommHandler.cpp
│   ├── BluetoothCommHandler.h # 蓝牙处理器
│   ├── BluetoothCommHandler.cpp
│   ├── TcpCommHandler.h       # TCP处理器（预留）
│   └── TcpCommHandler.cpp
├── CommViewModel.h
└── CommViewModel.cpp
```

**核心设计：**
```cpp
// CommManager.h
class CommManager : public QObject
{
    Q_OBJECT
public:
    // 设置当前通讯方式
    void setCurrentHandler(const QString& type);
    
    // 统一接口
    CommConnectState getConnectState() const;
    bool sendData(const QByteArray& data);
    void close();
    
    // 获取当前处理器
    CommHandler* currentHandler() const;
    
signals:
    void connectStateChanged(CommConnectState state);
    void dataReceived(const QByteArray& data);
    
private:
    QMap<QString, CommHandler*> m_handlers;
    CommHandler* m_currentHandler;  // 多态指针
};

// 抽象基类
class CommHandler : public QObject
{
    Q_OBJECT
public:
    virtual CommConnectState getConnectState() const = 0;
    virtual bool sendData(const QByteArray& data) = 0;
    virtual void close() = 0;
    virtual bool open(const QVariantMap& params) = 0;
    virtual QString type() const = 0;
    
signals:
    void stateChanged(CommConnectState state);
    void dataReceived(const QByteArray& data);
    void errorOccurred(const QString& msg);
};
```

**CommViewModel 简化后：**
```cpp
class CommViewModel : public QObject
{
    Q_OBJECT
    Q_PROPERTY(CommManager* commManager READ getCommManager CONSTANT)
    
public:
    CommViewModel(QObject* parent = nullptr);
    
    Q_INVOKABLE void sendData(const QString& data);
    Q_INVOKABLE void clearReceiveData();
    
private:
    CommManager* m_commManager;  // 只需持有管理器
};
```

### 方案 B：工厂模式 + 依赖注入

```cpp
class CommFactory
{
public:
    static CommHandler* create(const QString& type);
};

// 使用
CommHandler* handler = CommFactory::create("serial");
handler->open(params);
```

## 三、重构步骤

### 第一阶段：目录结构
1. 创建 `ViewModel/Comm/` 目录
2. 移动串口、蓝牙相关代码到独立文件

### 第二阶段：抽象接口
1. 定义 `CommHandler` 基类
2. 实现 `SerialCommHandler`、`BluetoothCommHandler`

### 第三阶段：通讯管理器
1. 实现 `CommManager` 类
2. 管理多个 Handler 的生命周期
3. 处理 Handler 切换逻辑

### 第四阶段：简化 CommViewModel
1. 移除直接的通讯参数属性
2. 通过 `CommManager` 统一操作
3. 保留数据发送/接收处理逻辑

### 第五阶段：界面适配
1. 更新 `SettingView.qml` 动态加载不同配置界面
2. `CommManager` 提供 `Q_INVOKABLE` 方法供 QML 调用

---

## 四、优势对比

| 项目 | 当前方案 | 重构后方案 |
|------|---------|-----------|
| 新增通讯方式 | 修改 CommViewModel | 新增 Handler 类 |
| 代码耦合度 | 高（集中在一个文件） | 低（职责分离） |
| 测试难度 | 难以单元测试 | Handler 可独立测试 |
| 可读性 | 随功能增加降低 | 保持良好 |
| 扩展性 | 需要修改大量代码 | 新增文件即可 |

## 五、接口设计

### Handler 接口规范

```cpp
class CommHandler : public QObject
{
    Q_OBJECT
    
public:
    // 类型标识
    virtual QString type() const = 0;  // "serial", "bluetooth", "tcp"
    
    // 连接管理
    virtual bool open(const QVariantMap& params) = 0;
    virtual void close() = 0;
    virtual CommConnectState getConnectState() const = 0;
    
    // 数据通信
    virtual bool sendData(const QByteArray& data) = 0;
    
    // 配置获取
    virtual QVariantMap getCurrentParams() const = 0;
    
signals:
    void connectStateChanged(CommConnectState state);
    void dataReceived(const QByteArray& data);
    void errorOccurred(const QString& error);
};
```

### CommManager 接口

```cpp
class CommManager : public QObject
{
    Q_OBJECT
    Q_PROPERTY(QStringList availableTypes READ getAvailableTypes)
    Q_PROPERTY(QString currentType READ getCurrentType)
    Q_PROPERTY(CommConnectState connectState READ getConnectState)
    
public:
    // Handler 管理
    void registerHandler(CommHandler* handler);
    void setCurrentHandler(const QString& type);
    
    // 统一操作
    bool open(const QVariantMap& params);
    void close();
    bool sendData(const QByteArray& data);
    
    // 获取可用 Handler 列表
    QStringList getAvailableTypes() const;
    
signals:
    void currentTypeChanged(const QString& type);
    void connectStateChanged(CommConnectState state);
    void dataReceived(const QByteArray& data);
    void errorOccurred(const QString& error);
};
```

## 七、注意事项

1. **向后兼容**：重构过程中保持现有接口不变
2. **信号转发**：`CommManager` 需要转发各 Handler 的信号
3. **生命周期**：`CommManager` 负责 Handler 的创建和销毁
4. **线程安全**：数据发送/接收可能涉及跨线程调用
